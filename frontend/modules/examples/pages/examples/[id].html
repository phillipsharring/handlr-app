2) Using Route Params with HTMX

You can pass route parameters to HTMX requests using hx-vals. The page enhancer (see below) automatically injects the ID into the data-example-id attribute, which HTMX can use.

4) Implementation Details

This example demonstrates the full dynamic routing implementation:

Dev Server (vite.config.js)

  • Detects [param] patterns in file paths
  • Matches URL segments to templates
  • Renders the correct template dynamically

Build (build-pages.mjs)

  • Preserves [param] in output paths
  • Generates dist/examples/[id]/index.html
  • One static file for all dynamic URLs

CloudFront (cloudfront-function.js)

  • Rewrites /examples/abc/
  • To /examples/[id]/index.html
  • Serves the static template file

Client-side (app.js)

  • getRouteParams(pattern) helper
  • Extracts params from URL
  • Makes them available to HTMX/JS

Code Example

Here's how the page enhancer for this example works (see app.js): 

// In initEnhancers() function in app.js:
const exampleDetailPage = root.querySelector('#load-example-btn');
if (exampleDetailPage) {
  // Extract the ID from the current URL
  const params = getRouteParams('/examples/[id]/');
  const id = params.id || 'unknown';

  // Make the ID available to HTMX via hx-vals
  exampleDetailPage.setAttribute('hx-vals',
    JSON.stringify({ id: id })
  );

  // Also update the display elements
  document.getElementById('current-id')?.textContent = id;
  // ...etc
}
