Home

Quick demos for boosted navigation, GET JSON templates, and JSON-encoded forms.

1) Boosted navigation (full-page URLs, HTMX swaps #app)

Click a link below. It should update the URL and replace only the #app content (not reload the whole page).

Go to Page 1 (boosted) Go to Page 2 (boosted)

2) Plain JS behavior (delegated binding survives swaps)

This is a simple JS hook in app.js (no HTMX request). It’s delegated so it keeps working after HTMX swaps.

3) GET JSON + client-side template (Handlebars)

HTMX fetches JSON from /api/examples/hello, then client-side-templates renders it using the Handlebars template below.

4) JSON-encoded form (POST) + client-side template

The json-enc extension sends form fields as JSON to /api/examples/echo. The response is JSON and is rendered via the template below.

(Server response will appear here.)

5) Modal + HTMX (swap into the global modal)

This reuses /api/examples/hello, but targets the global modal’s content area. When the swap completes, app.js auto-opens the modal.

6) Toast + HTMX (swap into the global toaster)

Calls /api/examples/toast which returns JSON like {"status":"success","message":"Saved!"}. The response is rendered via the toast template and the toast auto-hides after ~5s. Use the query string to control the status.

7) Form error handling examples

Three examples demonstrating the standardized error response system: field errors, invariant errors, and success.

Field Errors

Shows inline validation errors on form fields.

Invariant Error

Shows a business rule violation (no field errors).

Success

Shows a successful form submission.

8) Confirm dialog (global modal)

Uses App.ui.confirm.open(). Escape/click-outside cancels. Confirm runs an action (here: an alert).

9) Click Burst Effect

Click anywhere in the grey box below to see a burst effect. The burst is a gold/yellow circle that fades in, grows, and fades out (~0.5s).

Click me!

Add data-click-burst to any element. Optional data attributes: data-click-burst-color, data-click-burst-duration, data-click-burst-start-size, data-click-burst-end-size.

10) Takeover modal (fullscreen)

A fullscreen modal with blurred backdrop. No header/footer - the template provides its own UI. Clicking backdrop does not close it; only explicit close buttons work. Escape still works.
